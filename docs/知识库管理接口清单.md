# FinRAG 知识库管理接口清单（正式版，MVP）

**版本**：v1.0  
**状态**：正式版  
**说明**：以下接口用于知识库管理页面和后端联调，目标是最小必要、演示可用，并与现有问答接口兼容。

## 1. 设计原则

- 优先复用现有数据结构与检索链路
- 后端负责业务计算，前端只负责展示与触发
- 接口返回结构应适合页面直接渲染
- 失败信息必须可见，但不能阻塞其他文档处理

## 3. 接口列表

### 3.1 获取知识库概览

**GET** `/api/kb/overview`

用途：
- 页面顶部概览卡片

返回示例：

```json
{
  "total_documents": 40,
  "total_chunks": 9303,
  "last_import_at": "2026-05-13T19:06:00Z",
  "last_reindex_at": "2026-05-13T19:07:00Z",
  "status": "ready"
}
```

### 3.2 获取文档列表

**GET** `/api/kb/documents`

查询参数：
- `q`：关键词搜索
- `company`：公司筛选
- `doc_type`：文档类型筛选
- `status`：状态筛选
- `page`：分页页码
- `page_size`：分页大小

用途：
- 文档列表表格

返回示例：

```json
{
  "total": 40,
  "documents": [
    {
      "doc_id": "doc-001",
      "title": "贵州茅台 2025 年报",
      "company": "贵州茅台",
      "doc_type": "financial_report",
      "date": "2026-04-17",
      "source": "600519SH_moutai_annual_report_FY2025_2026-04-17_cninfo.pdf",
      "source_path": "backend/app/data/raw/extracted/reports/600519SH_moutai_annual_report_FY2025_2026-04-17_cninfo.md",
      "chunk_count": 240,
      "status": "active"
    }
  ]
}
```

### 3.3 获取文档详情

**GET** `/api/kb/documents/{doc_id}`

用途：
- 文档详情抽屉或侧栏

返回内容：
- 文档元数据
- 原始来源路径
- chunk 列表摘要
- 导入状态
- 错误信息（如有）

### 3.4 上传资料

**POST** `/api/kb/upload`

Content-Type：`multipart/form-data`

字段：
- `files[]`：一个或多个文件（PDF / MD / TXT）
- `collection_name`：知识库集合名
- `company`：可选默认公司
- `doc_type`：可选默认文档类型
- `date`：可选默认日期

用途：
- 将文件放入 raw 目录

返回示例：

```json
{
  "uploaded": 3,
  "saved_paths": ["backend/app/data/raw/manual/reports/a.md"]
}
```

### 3.5 开始导入

**POST** `/api/kb/import`

请求体示例：

```json
{
  "collection_name": "reports",
  "source_dir": "backend/app/data/raw/extracted/reports",
  "processed_dir": "backend/app/data/processed",
  "rebuild_index": true,
  "default_company": "未知",
  "default_doc_type": "research_report",
  "default_date": "unknown"
}
```

用途：
- 触发 raw → documents/chunks 的导入
- 可选重建索引

返回示例：

```json
{
  "job_id": "job-20260513-001",
  "status": "running"
}
```

### 3.6 查询导入任务

**GET** `/api/kb/import-jobs/{job_id}`

用途：
- 页面轮询导入状态

返回示例：

```json
{
  "job_id": "job-20260513-001",
  "status": "completed",
  "total_files": 40,
  "success_count": 40,
  "fail_count": 0,
  "reindex_status": "completed",
  "error_messages": []
}
```

### 3.7 重新导入单篇文档

**POST** `/api/kb/documents/{doc_id}/reimport`

用途：
- 对单篇文档触发重新导入

返回示例：

```json
{
  "job_id": "job-20260513-002",
  "status": "running"
}
```

### 3.8 停用/删除文档

**DELETE** `/api/kb/documents/{doc_id}`

建议语义：
- 默认软删除或停用，不直接删除原始文件

返回示例：

```json
{
  "doc_id": "doc-001",
  "status": "disabled"
}
```

### 3.9 重新构建索引

**POST** `/api/kb/reindex`

用途：
- 仅重建 BM25 / vector 索引，不重新导入 raw 文件

返回示例：

```json
{
  "status": "running"
}
```

## 4. 与现有接口的关系

### 保持不变

- `GET /api/documents`
- `POST /api/query`
- `POST /api/preview-rewrite`

### 建议关系

- 知识库管理页面优先使用 `/api/kb/*` 接口
- 现有文档库与问答页面继续走原有接口
- 知识库导入完成后，原有文档列表和问答链路应自然读取新数据

## 5. 错误约定

建议统一返回：

```json
{
  "error": {
    "code": "IMPORT_FAILED",
    "message": "chunk generation failed for one file",
    "details": []
  }
}
```

常见错误码建议：

- `INVALID_INPUT`
- `FILE_TOO_LARGE`
- `IMPORT_FAILED`
- `REINDEX_FAILED`
- `JOB_NOT_FOUND`
- `DOCUMENT_NOT_FOUND`

## 6. 联调建议

前端联调时，优先实现以下闭环：

1. 拉取概览与文档列表
2. 上传文件
3. 创建导入任务
4. 轮询任务状态
5. 显示导入结果
6. 重新导入或停用文档

如果后端接口尚未全部实现，前端可先用 mock 适配层保持页面完整，再逐步切换到真实接口。

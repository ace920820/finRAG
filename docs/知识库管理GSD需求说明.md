# FinRAG 知识库管理 GSD 需求说明

**用途**：作为 GSD 工作流的需求边界说明  
**范围**：只覆盖后端功能开发与前后端联调，不包含前端页面设计与视觉实现

## 1. 背景

前端 React 页面会由其他团队实现并交付给当前项目。当前阶段的重点不是重新设计界面，而是让后端能力、导入管线和联调接口先稳定下来，方便后续快速接入前端页面。

## 2. 本期目标

本期只做两件事：

1. 完成知识库管理相关的后端能力开发
2. 完成前后端联调所需的接口、数据结构和测试验证

## 3. 需求边界

### 3.1 包含内容

- 知识库概览接口
- 文档列表与详情接口
- 上传接口
- 导入任务接口
- 单篇重新导入
- 文档停用/删除
- 索引重建
- 与现有问答接口的数据联通
- 后端测试与联调验证

### 3.2 不包含内容

- 前端页面设计
- 前端组件开发
- UI 视觉规范
- 动效设计
- 多用户权限
- 审批流
- 复杂的后台运营能力

## 4. 开发重点

### 4.1 后端开发

后端需要完成以下能力：

- 接收并保存上传文件
- 解析 PDF / MD / TXT
- 生成 documents / chunks 数据
- 维护导入任务状态
- 提供文档列表和详情查询
- 支持停用、重新导入和重建索引

### 4.2 联调准备

联调阶段需要保证：

- 接口路径和字段命名稳定
- 返回值适合前端直接渲染
- 失败信息清晰可见
- mock 和真实数据都能切换
- 现有问答接口不被破坏

## 5. 推荐执行顺序

1. 后端接口定义与实现
2. 导入与索引流程打通
3. 文档状态与错误处理补齐
4. 与前端页面联调
5. 基础测试与回归验证

## 6. 验收标准

- 前端页面能够调用知识库管理接口并完整展示数据
- 上传、导入、重建索引可用
- 文档停用和重新导入可用
- 问答接口仍可正常使用导入后的知识库
- 后端测试通过，联调问题可定位、可复现、可修复

# FinRAG 金融 QA Benchmark 设计

## 1. 目标

这份 benchmark 用于评估 FinRAG 在当前数据集上的金融问答能力，重点考察：

- 是否能从多公司、多年份、多类型金融文档中找对证据。
- 是否能正确读取 PDF 表格抽取出的数值。
- 是否能基于年报/研报做有边界的分析总结。
- 是否能在资料不足时拒答，而不是编造。
- 是否能生成带引用、结构化、可追溯的 Markdown 答案。

该 benchmark 面向小而高质量的人工评测，也可以后续转成 JSONL 自动评测集。

---

## 2. 当前数据集覆盖

当前 processed 数据集包含：

- 文档数：40 份。
- chunk 数：14,091 个。
- 表格事实数：1,558 条。
- 公司覆盖：宁德时代、贵州茅台、NVIDIA、台积电。
- 文档类型：年报、季报、半年报、10-K、10-Q、券商研报。
- 核心指标覆盖：`revenue`、`net_income`、`operating_income`、`eps_diluted`、`gross_profit`。

典型数据文件：

- `backend/app/data/processed/documents.json`
- `backend/app/data/processed/chunks.json`
- `backend/app/data/processed/table_facts.json`
- `backend/app/data/raw/tables/finrag-user-source-40/**/*.json`

---

## 3. 样本设计原则

每条样本包含：

- 用户问题：实际会问系统的问题。
- 问题类型：事实查找、表格数值、对比分析、风险总结、多文档综合、拒答。
- 标准答案要点：人工评测时必须覆盖的内容。
- 必须命中的 source chunk：理想情况下 Top5 evidence 或最终 citation 应命中的 chunk/table/fact。
- 可接受引用范围：允许命中的文档、页码、表格或 chunk 范围。
- 金融指标或数值校验点：用于判断数值是否准确。
- 人工评分标准：5 分制。

评分建议：

| 分数 | 标准 |
|---:|---|
| 5 | 证据命中、数值准确、引用正确、回答结构清楚，没有无依据扩展。 |
| 4 | 核心答案正确，证据或表达有小瑕疵。 |
| 3 | 找到部分相关证据，但遗漏关键数值/原因/边界。 |
| 2 | 方向相关但证据不可靠，或混淆年份/公司/指标。 |
| 1 | 答案大多错误或引用明显不支持。 |
| 0 | 编造、答非所问，或应拒答但没有拒答。 |

- NVIDIA 相关 10-K/10-Q/研报均可作为“当前资料范围”说明。
- 不接受没有来源的预测数值。

**金融指标或数值校验点**

- 不允许输出确定的 FY2028 data center revenue。

**人工评分标准**

- 5 分：明确拒绝确定预测，同时可给出有依据的边界性分析。
- 3 分：拒答但没有解释资料范围。
- 0 分：编造 FY2028 数值。

## 5. 推荐评测方法

### 5.1 检索质量评估

对每条样本记录：

- Top5 是否命中必须 chunk/table/fact。
- `citation_id` 是否指向支持答案的证据。
- 表格数值题是否命中 `table`、`table_row` 或 `table_fact`。

推荐指标：

- Recall@5：Top5 evidence 是否包含任一可接受证据。
- MRR：第一个正确证据出现在第几名。MRR 是离线评测指标，不是线上回答步骤。
- Evidence Precision@5：Top5 中有多少条是真正相关证据。

### 5.2 答案质量评估

每题人工 0-5 分，拆成四个维度：

| 维度 | 分值 | 说明 |
|---|---:|---|
| 证据命中 | 1 | 引用是否命中可接受证据。 |
| 数值准确 | 1.5 | 金融数值、单位、期间是否正确。 |
| 推理/总结 | 1.5 | 是否正确解释变化、风险或增长驱动。 |
| 边界控制 | 1 | 是否避免无依据扩展，拒答题是否拒答。 |

### 5.3 通过标准

建议 MVP 演示通过线：

- 平均答案分：`>= 4.0 / 5`
- 事实/表格数值题：`>= 4.5 / 5`
- 拒答题：必须 `>= 4 / 5`
- Recall@5：`>= 80%`
- 不允许出现高置信编造金融数值。

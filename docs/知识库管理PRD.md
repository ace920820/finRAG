# FinRAG 知识库管理 PRD（正式版，MVP）

**版本**：v1.0  
**状态**：正式版  
**适用范围**：FinRAG 演示项目的知识库管理能力  
**责任边界**：前端页面由外部 React 团队实现；本文档重点定义知识库管理页面的最小功能、后端能力与前后端联调边界。

## 1. 背景

FinRAG 当前已经具备问答检索链路、文档展示能力以及文档导入基础。为了让演示更完整，需要补充一个轻量的知识库管理页面，用于查看、导入和维护知识库中的文档。

本项目定位为演示型 MVP，因此不引入复杂的权限、审批、版本管理或协作编辑能力，只保留对演示最有价值、且便于后续联调的最小功能集。

## 2. 目标

本期目标是建立一个可用、可解释、可联调的知识库管理闭环，使系统具备以下能力：

1. 查看知识库总体概览
2. 查看和筛选文档列表
3. 上传 PDF / MD / TXT 资料
4. 触发导入与索引重建
5. 查看导入结果、失败原因和任务状态
6. 维护单篇文档的启用、停用和重新导入

## 3. 项目原则

- **最小必要**：只实现演示与联调所需的核心能力。
- **后端优先**：核心逻辑由后端承担，前端只负责展示与触发。
- **接口清晰**：接口返回结构应足够直接，便于 React 页面渲染。
- **兼容现有链路**：知识库管理能力应尽量复用现有问答与导入链路。
- **便于测试**：所有关键流程都要能通过 mock 或真实数据进行验证。

## 4. 使用场景

### 场景 1：查看知识库状态

用户打开知识库管理页，先看到文档总数、chunk 总数、最近导入时间和最近索引重建时间，快速判断当前知识库是否可用。

### 场景 2：导入新资料

用户上传一批 PDF、Markdown 或 TXT 文件，指定集合名和必要的默认元数据，系统创建导入任务并在后台完成处理。

### 场景 3：查看导入结果

用户在任务进行中查看成功数、失败数和失败原因；如果有某些文件失败，不影响其他文件继续导入。

### 场景 4：维护文档

用户可以对单篇文档执行重新导入，或者对不再需要的文档执行停用操作。

## 5. 范围说明

### 5.1 本期包含

- 文档概览
- 文档列表
- 文档详情

## 6. 功能需求

### 6.1 前端功能需求

#### F-1 知识库概览

页面顶部展示以下信息：

- 文档总数
- Chunk 总数
- 最近导入时间
- 最近索引重建时间
- 当前状态：正常 / 导入中 / 失败 / 未初始化

#### F-2 文档列表

列表至少展示以下字段：

- 文档标题
- 公司
- 文档类型
- 日期
- 来源
- chunk 数量
- 状态

支持以下筛选：

- 关键词搜索
- 公司筛选
- 文档类型筛选
- 状态筛选

#### F-3 文档详情

点击文档后，页面应展示：

- 文档元数据
- 原始来源路径
- chunk 摘要
- 导入状态
- 失败原因（如有）

#### F-4 资料导入

支持：

- 上传 PDF / MD / TXT
- 选择 collection_name
- 可选填写默认公司、默认文档类型、默认日期
- 提交导入任务

#### F-5 索引重建

支持：

- 导入完成后触发索引重建
- 单独触发索引重建

#### F-6 文档维护

支持：

- 重新导入单篇文档
- 停用/删除文档

#### F-7 任务状态

任务区应展示：

- 任务 ID
- 当前状态
- 成功数 / 失败数
- 错误信息
- 是否已完成索引重建

### 6.2 后端功能需求

#### B-1 资料接收

后端需接收并保存以下文件类型：

- PDF
- Markdown
- TXT

建议保存位置：

- `backend/app/data/raw/extracted/<collection-name>/`
- `backend/app/data/raw/manual/<collection-name>/`

#### B-2 导入解析

后端需将 raw 内容解析并生成：

- `documents.json`
- `chunks.json`

同时保留以下元数据：

- 标题
- 公司
- 文档类型
- 日期
- 来源
- 页码
- 原始路径
- collection 名称

#### B-3 确定性 ID

后端需保证：

- 相同输入重复导入时，`doc_id` 保持稳定
- 相同输入重复导入时，`chunk_id` 保持稳定

#### B-4 任务状态

后端需记录：

- 任务创建时间
- 任务开始时间
- 任务结束时间
- 成功数量
- 失败数量
- 错误列表
- 是否完成索引重建

#### B-5 索引重建

后端需支持：

- BM25 重建
- vector 重建
- 使用现有检索链路的索引文件格式

#### B-6 文档维护

后端需支持：

- 文档列表查询
- 文档详情查询
- 软删除/停用
- 重新导入单篇文档

#### B-7 联调兼容

后端需保证：

- 现有 `GET /api/documents` 保持可用
- 现有 `POST /api/query` 不改协议
- 知识库管理页面通过新接口完成管理操作
- 前端后续导入 React 页面后，可直接与这些接口联调

## 7. 验收标准

以下条件满足后，本期可以视为完成：

- 能查看知识库概览和文档列表
- 能上传新资料并完成导入
- 能触发索引重建
- 能查看导入成功/失败结果
- 能停用或重新导入单篇文档
- 现有问答接口仍可使用导入后的知识库
- 前端无需重构主问答页面即可接入知识库管理能力

## 8. 交付优先级

推荐实现顺序：

1. 文档概览与列表
2. 上传与导入
3. 导入状态与失败展示
4. 索引重建
5. 文档停用与重新导入

## 9. 备注

- 前端页面由 React 团队实现，本文档不定义具体视觉稿。
- GSD 后续工作重点应放在后端功能开发与前后端联调，而不是前端设计。

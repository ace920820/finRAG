## Milestone Summary

**Key Decisions:**

- Keep PDF extraction in `pdf2md/`, and keep FinRAG schema import in backend code.
- Canonical raw corpus location is `backend/app/data/raw/`.
- Runtime APIs continue reading existing `backend/app/data/processed/*.json` and `backend/app/data/index/*` contracts.
- Use mock providers by default in tests so ingestion validation does not require Bailian credentials.
- Preserve fixture/demo fallback path while allowing real corpus import.

**Issues Resolved:**

- Removed Elite Daily naming/layout assumptions from the FinRAG extraction path.
- Added a real PDF/Markdown → documents/chunks → indexes pipeline.
- Fixed stale vector-index risk by supporting force rebuild.
- Stabilized offline tests against local `.env` credentials by forcing mock providers in tests and normalizing blank secrets.
- Imported and indexed 40 real PDFs across 宁德时代、贵州茅台、NVIDIA、台积电.

**Issues Deferred:**

- OCR for scanned PDFs.
- Production ingestion UI/background jobs.
- Large-scale incremental indexing.
- Knowledge graph extraction.
- Advanced financial table reconstruction.

**Technical Debt / Follow-up Candidates:**

- The generated corpus and index files are large and may need a data artifact strategy before frequent commits.
- The `pdf2md` directory is currently a nested/untracked project in the parent repo and may need submodule/vendor policy.
- Metadata inference is filename/path based and may need a formal metadata manifest for higher-quality corpora.

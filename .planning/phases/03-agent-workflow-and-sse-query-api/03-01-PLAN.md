<threat_model>
## Threat Model

### Assets
- User financial queries.
- Retrieved financial evidence chunks.
- Bailian/DashScope API credentials.
- Citation integrity.

### Threats
- **High:** Generated answer includes unsupported claims or loses citation grounding.
  - Mitigation: prompt builder must include only top reranked evidence and mock generation must cite retrieved chunks.
- **High:** Tests or defaults accidentally call live model APIs.
  - Mitigation: keep `text_provider=mock` as default and mock tests independent of env/API keys.
- **Medium:** Query workflow mutates retrieval/rerank contracts needed by Phase 4.
  - Mitigation: consume existing schemas and add new agent modules instead of refactoring Phase 2 services.
- **Medium:** Provider implementation logs or commits secrets.
  - Mitigation: do not print API keys; do not create `.env`; use existing settings only.

### Blocking Requirement
No live API key is required for any automated test or local default workflow.
</threat_model>

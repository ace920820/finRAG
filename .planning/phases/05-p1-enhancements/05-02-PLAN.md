## Threat Model

### Assets
- User typing experience.
- Backend preview endpoint load.
- Main demo query flow.

### Threats
- **High:** Preview requests spam the backend while typing.
  - Mitigation: debounce requests and abort stale in-flight calls.
- **High:** Preview UI drifts away from the stable design.
  - Mitigation: reuse the existing preview area and placeholder layout.
- **Medium:** Preview failures block normal query sending.
  - Mitigation: preview errors should degrade silently or show a subtle fallback, never block submit.

### Blocking Requirement
Do not alter the main `POST /api/query` send flow or introduce a new design system.
</threat_model>

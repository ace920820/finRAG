<threat_model>
## Threat Model

### Assets
- Existing backend schema compatibility.
- Deterministic imported corpus output.
- Demo fixture safety.

### Threats
- **High:** Generated JSON does not match `Document`/`Chunk` schemas.
  - Mitigation: instantiate Pydantic models in tests and/or importer validation before writing.
- **High:** IDs change between identical imports.
  - Mitigation: derive `doc_id` and `chunk_id` from stable source identity/content hash.
- **Medium:** Metadata is lost from Phase 6 frontmatter.
  - Mitigation: preserve frontmatter fields in document fields and chunk metadata.
- **Medium:** Import core overwrites existing demo fixtures during tests.
  - Mitigation: use explicit output dirs and temp directories in tests.
- **Low:** Chunking becomes over-engineered.
  - Mitigation: implement simple deterministic paragraph/length chunking with page marker awareness.

### Blocking Requirements
- Do not modify frontend code.
- Do not require live Alibaba Bailian credentials.
- Do not remove or rewrite `scripts/seed_data.py`.
</threat_model>

<threat_model>
## Threat Model

### Assets
- Extracted Markdown source fidelity.
- Source-file traceability for future corpus import.
- Safe repeated batch runs.
- Existing `pdf2md` behavior for current users.

### Threats
- **High:** Elite Daily-specific filename or season assumptions still block generic financial PDFs.
  - Mitigation: add a generic FinRAG mode/adapter that accepts arbitrary PDF names and uses robust fallback titles.
- **High:** Traceability is lost before Phase 7 can build document/chunk JSON.
  - Mitigation: include source path/name, normalized title, page count, extractor, status, and content hash in frontmatter or manifest.
- **High:** One bad PDF aborts a full import batch.
  - Mitigation: preserve per-file exception capture and failure manifest entries.
- **Medium:** Existing idempotency regresses.
  - Mitigation: keep hash-based skip semantics and add a focused test.
- **Medium:** Adapter change becomes a broad rewrite.
  - Mitigation: introduce the smallest mode/module possible and reuse current extraction/output primitives.

### Blocking Requirements
- Do not add OCR.
- Do not generate FinRAG `documents.json`, `chunks.json`, or indexes in Phase 6.
- Do not commit `.env`, API keys, AppleDouble files, or unrelated generated backend artifacts.
</threat_model>

<threat_model>
## Threat Model

### Assets
- Backend/frontend contract compatibility.
- Frontend build stability.
- Demo reliability when backend is unavailable.

### Threats
- **High:** SSE parser drops partial frames or misorders events.
  - Mitigation: implement a small frame parser with unit tests using split sample frames.
- **High:** Backend field names leak into UI components and make future changes brittle.
  - Mitigation: centralize all mapping in `frontend/src/api/finrag.ts`.
- **Medium:** Vite proxy is missing or hard-codes production assumptions.
  - Mitigation: configure dev-only `/api` proxy to `http://localhost:8000`.
- **Medium:** Adapter tests add heavyweight test framework setup.
  - Mitigation: prefer pure functions that can be typechecked; only add lightweight tests if existing tooling supports it cleanly.

### Blocking Requirement
Do not add real API keys, production URLs, or frontend redesign work.
</threat_model>

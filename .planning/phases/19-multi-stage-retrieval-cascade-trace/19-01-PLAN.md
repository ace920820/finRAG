<threat_model>
## Threat Model

- **[high] Contract drift in SSE clients** — Use only additive fields and preserve existing event names/order.
- **[high] Trace lying about execution** — Build stages from actual route/filter/count/error variables, not from expected happy-path assumptions.
- **[medium] Debug leakage** — Keep trace metadata to counts, route/filter summaries, method names, and fallback reasons; do not expose provider secrets or raw API responses.
- **[medium] Runtime regressions** — Avoid additional network calls or extra retrieval passes just to build trace.
- **[medium] Scope creep into later phases** — Do not implement evidence compression, iterative retrieval, or hierarchy changes.
</threat_model>

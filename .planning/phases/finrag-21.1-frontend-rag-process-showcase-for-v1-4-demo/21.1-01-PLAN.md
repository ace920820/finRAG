<threat_model>
<assets>
- Accurate representation of backend RAG process artifacts.
- User trust in source-grounded answers and citations.
- Demo clarity during interview walkthroughs.
- Existing chat/citation/KB frontend functionality.
</assets>

<threats>
- **Medium: Misleading demo data** if the UI fabricates hierarchy or future backend outputs.
  Mitigation: render only real payload data; Phase 22 hierarchy appears only from actual evidence/citation metadata or `hierarchy_drill_down` stages.
- **Medium: Type drift** if frontend payload types diverge from backend events.
  Mitigation: keep fields optional and validate with TypeScript/build checks.
- **Medium: UI overflow** from long generated queries, filters, or evidence titles.
  Mitigation: wrap long text, keep panels compact, and avoid fixed-width text assumptions.
- **Low: Citation regression** while adding new inspector sections.
  Mitigation: preserve existing rerank document IDs and citation click path.
</threats>

<security_acceptance>
- Frontend does not expose provider secrets, environment variables, or hidden backend config.
- Frontend does not add new document access paths.
- Debug/process metadata is limited to data already returned by `/api/query`.
</security_acceptance>
</threat_model>

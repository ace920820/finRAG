<threat_model>
## Threat Model

### Assets
- API contract consumed by the future React frontend.
- Local demo data and processed JSON files.
- Provider/API credentials that may be configured in later phases.

### Threats
- **Medium:** Accidental exposure of future API keys through config defaults or test fixtures.
  - Mitigation: config must read keys from env only and never write secrets into fixtures or logs.
- **Low:** Over-permissive CORS defaults may be copied into later demo code.
  - Mitigation: keep CORS explicit and local-development oriented if added.
- **Low:** Schema drift breaks frontend integration.
  - Mitigation: define schemas centrally and test serialized payloads.

### Blocking Requirement
No secrets, API keys, or real credentials may be committed. Phase 1 must run without credentials.
</threat_model>

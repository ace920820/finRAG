<threat_model>
## Threat Model

### Assets
- Local index artifacts.
- Fixture document/chunk metadata used for citations.
- Retrieval output consumed by future frontend/SSE integration.

### Threats
- **Medium:** Index files become stale relative to processed fixtures.
  - Mitigation: build script should rebuild deterministically and tests should validate expected fixture retrieval.
- **Medium:** Retrieval output loses metadata needed for citations.
  - Mitigation: map hits through `Chunk` metadata into `RetrievalResultItem` fields.
- **Low:** Debug or index code reads arbitrary files.
  - Mitigation: restrict index load/build paths to configured backend data directories.

### Blocking Requirement
Retrieval tests must not depend on live embedding APIs; use deterministic mock embeddings.
</threat_model>
